# Отчет по практической работе: Дополнительные возможности Docker

## Цель работы
Изучение инструментов Docker для управления контейнерами, мониторинга ресурсов, а также сохранения и восстановления состояния контейнеров (export/import).

---

## Ход работы

### 1. Вывод логов контейнера в файл
Была изучена работа с логированием в Docker. Запустил контейнер, который генерирует логи в стандартный вывод. Затем сохранил эти логи в отдельный файл на хост-машине для последующего анализа.

**Использованная команда:**
```bash
docker logs practice-container-1 > /tmp/container_logs.txt
Результат: Логи успешно сохранены и прочитаны из файла.

2. Проверка docker-stats
Изучен инструмент мониторинга потребления ресурсов в реальном времени. Запустил контейнер в фоновом режиме и проверил его статистику потребления CPU и оперативной памяти с помощью встроенной утилиты.

Использованная команда:

Bash

docker stats --no-stream practice-container-2
Результат: Получены данные о текущем потреблении ресурсов контейнером.

3. Ограничение ресурсов (CPU и Memory)
Протестирован механизм ограничения ресурсов для контейнеров. Запустил контейнер с жестким лимитом по памяти (256Mb) и процессору (0.5 CPU). В процессе работы контейнера динамически обновил лимиты памяти до 512Mb без его перезагрузки.

Использованные команды:

Bash

docker run -d --name practice-limited --memory=256m --cpus=0.5 practice-image:1.0 60
docker update --memory=512m practice-limited
Результат: Лимиты успешно применились, что подтверждено через docker stats.

4. Экспорт контейнера в tar-архив
Выполнен экспорт файловой системы контейнера в переносимый tar-архив. Это позволяет создать снимок состояния контейнера для его переноса или резервного копирования.

Использованная команда:

Bash

docker export practice-export > /tmp/container_export.tar
Результат: Архив создан, проверен его размер и содержимое.

5. Импорт контейнера из tar-архива
Выполнено восстановление образа из ранее созданного архива. На основе восстановленного образа был успешно запущен новый контейнер, который продолжил работу с сохранением файловой структуры оригинала.

Использованная команда:

Bash

docker import /tmp/container_export.tar restored-practice:1.0
Результат: Образ успешно импортирован, контейнер запущен и функционирует корректно.

Вывод
В ходе практической работы были освоены навыки управления жизненным циклом контейнеров, их диагностики через логи и статистику, а также методы управления ресурсами и миграции контейнеров через механизмы export/import.
